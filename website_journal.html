Day 3 - May 24, 2017

Most of my learning today will be through link(7). I thought it would be easier to learn HTML and CSS through an organized lesson by codeacademy. I learned the title should always be in the the head element, so I corrected my code and moved the title element.

I then learned about two types of lists unordered and ordered lists. Unordered are bullet point lists and ordered lists have numerical associations. The "ul" and "ol" elements can't interpret raw text, but it can organize "li" elements together.

In order to add hyperlinks connecting my webpage to the rest of the internet, I need to use "a" (aka anchor element). In addition to using the "a" element, I need to add an attribute href, which lives in the openning tag. There's two parts of an attribute: name of attribute and value of attribute. The name in this case is href, and the value is the web address string. Cool stuff. The anchor element also has a target attribute, which opens the link in a new window (new tabs in modern browsers). The value for target is "_blank". The cool thing about the an anchor element is that you can wrap any other element with "a" and it becomes a link of some form.

The img element are self closing elements. Meaning it only has an openning tag that ends with />. I have used the src, alt, and style attributes. The src is like href but for an image, which could be an address to the image or an image in a local directory. alt is the the text that shows if the image doesn't load properly. style is how the image should be displayed.

CSS is used to modify the text of HTML to become visually appealing. According to Code Academy, it can be used to modify colors, font types, font sizes, shadows, images, element positioning, and more. The inline CSS, as I learned about in a previous day, must be placed in the head element. The general syntax of CSS is an element selector followed by curly brackets and it's contents. For example, all elements that are selected will have these values. * is the universal selector

selector {
property: value;
}

It's generally good practice to separate HTML and CSS code into two separate folders even though CSS can be incorporated into the head element of HTML. The "link" element is used to associated the two files together. Similar to a #include in C and C++ but different. It must be included in the head element that requires the following attributes: href, type, rel. href is the path to the CSS file. type is the type of document, which is text/css in this case. rel is the relationship between the two, in this case it's a "stylesheet".

Day 4 - May 25, 2017

Learning about colors for CSS. There's 3 different ways to specify the color values: using named colors (147 named colors), using rgb values (each ranging from 0-256: adobe's color theme software), or hex values, which are the same as rgb values but different representation. to represent rgb use rgb(red, green, blue).

Aside from these three representations, there's also HSL (Hue, Saturation, and Lightness). Hue is the color wheel from 0 to 360. Saturation is the amount of gray in a color from 0% to 100% where it represents the shade of gray and 100% is full saturation. Lightness is the amount of white in a color. Like saturation, it ranges from 0% to 100%, but 0% is black, 100% is white, and %50 is normal. To represent hsl we use hsl(hue, saturation, lightness).

Day 5 - May 26, 2017

Colors have another property in addition to the rgb intensities called an alpha channel. The alpha channel is used for transparency. The lower the value the more transparent, the greater the value more opaque. To specify this value in addition to rgb, you use rgba(). The alpha channel ranges from 0 to 1 for all real numbers. According to codeacademy, we can declare both rgb and rgba in case the older browser doesn't support the alpha channel. The fallback color should precede rgba color. hsl function also supports the alpha opacity channel.

After colors, I learned about fonts. serif and sans-serif are two types of fonts. Serif has extra strokes on the ends of the characters whereas sans-serif don't, which creates a cleaner look. When selecting the font-family property, the developer can specify multiple fonts separated by commas. The left most font is used and each one to the right are fallbacks if the browser doesn't support the font.

Google fonts has a ton of fonts that web developers can import to design their websites. To import the font, you will need to use the link element where the href value is equal to: "https://fonts.googleapis.com/css?family=Barrio" where Barrio is replaced with the font you want, and spaces are replaced with '+'. The type value is "text/css" and the rel value is "stylesheet". The element should be placed in the head element. I experimented with the positioning of the element with respect to the link element containing the style sheet that uses the font, and it appears that it doesn't matter if the font is included before or after the css file unlike C or C++.

According to stack overflow's explanation, there is another way to import the font with the @import method, but in this case, it must be the first thing in the file.

According to this stack overflow thread "single huge .css files vs multiple smaller specific .css files", there's an efficiency difference between one css file vs multiple css files. Having one CSS file is more efficient because there's less http requests. Based on the in-depth explanation on quora about the same topic, CSS files are cached when it gets loaded. That way if the same file is needed again by another page, it can use the cached version instead of loading a new file. This explains a lot of the problems I've been having where I update the CSS file, and it doesn't affect my page when I refresh it. The page is still using the old cached CSS file.

Day 8 - June 22, 2017

I added a simple Navigation Bar by following this youtube tutorial. This guy did a very good job at explaining/demonstrating the process. For the time being I will keep it how it is to observe the visual affects it has on the website. When I get the change later I will be changing the design and make it more personalized.

There appears to be an unintentional affect. There's some sort of margin around the Navigation Bar. When I remove all other styling from the css file the margin is still around the Bar. I'm not quite sure where this problem is coming from, but I will need to investigate or ask for help from someone.

I decided to set margins to 0px and it appears to have an affect to the Navigation Bar. It partially closed the gap between the top of the webpage and the top of the Bar. Perhaps there are uninitialized values that I didn't account for, which doesn't make sense. 0px should be the default value if the margin isn't specified.

It seems that I fixed the problem. Here's what I have added. left: 0px and top: 0px. It seems that these two properties don't have the value of 0px, which makes sense why the Navigation Bar is offset from the sides and top of the screen. I tested this out when I added the two properties: position and width. I added the position so I can keep it fixed to a specific location as the rest of the screen scrolls. I added width to be 100% because adding position caused the Navigation Bar to shrink to only the necessary size around the two links: Home and Web Journal. Now I have an ideal simple Navigation Bar. Also the margin 0px that I added earlier seems to be required. Without it the top of the Bar still offsets slightly. This seems strange since I had the top property set to 0px.

I noticed that adding the position to be fixed caused the initial content to hide behind the navigation bar. I searched for a solution and came across a stack overflow explanation. I ended up adding a padding-top: 40px. This pushed the content down. This is an interesting solution, now the bottom of the Navigation Bar acts like the top of the body.

Day 9 - June 24, 2017

Today I decided to make the modifications to the Navigation Bar, Nav Bar for short. I will be adding some icons to the bar to make it a little more fancy. I found some icons at the iconarchive.com. There's all sorts of icons that you can use. I'll be picking simple ones so that it's not too overwhelming to the viewers. I will also be adding on to the implementation of the Nav Bar by building onto what the YouTube video suggested.

To start off, I need to change the implementation of the Nav Bar because using a list is a bit restrictive. With a list, All my items must be clumped together, I can't place one item to the far left and all the other items to the center. To fix this problem, I created a container class outside of the list. The container becomes my Nav Bar and the list becomes an item. In the future, I will be able to add a unique icon that represents my website in addition to the navigation links and any other items I want to add for ease of access. By adding this container, I needed to make some modifications to the positioning in the style file. Instead of a fixed position for the list, I changed it to absolute. According to w3school, this based the positioning of the NavBar relative to its first positioned ancestor element. When I add properties such as bottom, left, top, right. It will stay contained within the container element and not the body element.

I accidentally zoomed in on a page and the items started moving out of place. I will have to fixed this problem a different time... Turns out the problem was because I used percentage values for the height of the Nav Bar and the padding-top for the body. When you zoom in, the size of the elements also change to maintain the percentage values.

The icon idea was scrapped because it complicates the design. For user experience, icons are also less descriptive than words. Using words are makes it easier to navigate the website. I am now back to square one figuring out how to position the items. I might follow the design of StackOverflow's Nav Bar.

It appears that I went full circle, I decided to remove the container all together.
